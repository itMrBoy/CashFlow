# CashFlow Vibe - Sprint 1 需求文档 (PRD)

## 1. 产品概述

纯本地运行的个人财务与订阅管理工具，采用新极简粗野风格（Neo-Brutalism）视觉风格，提供极简、高频的资金流转记录体验。

## 2. 核心页面与功能模块

### 2.1 首页看板 (Dashboard)

- **核心指标**：展示周期内可用结余、总收入、总支出、固定支出（订阅）。
- **预算进度**：粗边框进度条展示已用预算比例。
- **周期选择器 (Period Selector)**：
  - 点击标题下方的“📅 本月 (THIS MONTH)”药丸按钮，唤出半屏抽屉 (Bottom Sheet)。
  - 支持快捷选项（本月、上月、近30天、今年）和自定义日期范围筛选。看板数据需根据所选周期联动刷新。

### 2.2 收支流水 (Transactions)

- **状态总览**：展示当前筛选周期内的“总支出”与“总收入”。
- **流水列表**：按时间倒序排列，展示分类 Icon、标题、副标题（如有）、金额（收入绿色，支出黑色），编辑、左滑删除等功能。
- **高级筛选 (Filter Modal)**：
  - 包含收发类型、分类网格、日期范围。
  - 筛选应用后，列表顶部需展示实体票根样式的“已应用筛选条件”（支持点击 X 移除）。
- **记一笔弹窗 (Add Record Modal)**：
  - 全屏弹窗，支持“支出”与“收入” Tab 切换。
  - 包含标题、金额、分类标签。
  - 当选中特定分类（如“礼金”）时，动态显示“对象”输入框。

### 2.3 周期订阅 (Subscriptions)

- **状态总览**：展示每月平均订阅花费。
- **订阅列表**：展示服务名称、周期、距离下次扣费天数（小于等于 3 天时，右侧标签标红警告），编辑、左滑删除等功能。
- **添加订阅弹窗 (Add Subscription Modal)**：
  - 全屏弹窗，录入名称、金额、周期、结算日、分类。
  - **核心联动**：底部包含“立即记一笔支出”开关。开启后，不仅保存订阅规则，还会在流水中自动生成一笔当天的支出记录。

### 2.4 设置 (Settings)

- **功能描述**：备份、恢复、清空数据。
- **核心功能**：
  - **备份**：支持导出json文件，并保存到本地。
  - **恢复**：支持从json文件恢复数据。
  - **清空数据**：支持清空所有数据。

### 2.5 场景化新手引导 (Contextual Micro-Onboarding)

- **功能描述**：通过高亮遮罩和自定义提示框，在用户首次进入特定页面且满足特定条件时，提供沉浸式的操作指引。避免全局漫游带来的打扰和跨页面的定位 Bug。
- **视觉规范**：完全继承 Neo-Brutalism 风格。遮罩层极深 (Opacity 0.8)；提示框 (Tooltip) 必须具有极粗纯黑边框、荧光绿/亮黄色底色及硬阴影；按钮样式需保持高对比度。
- **分步触发策略**：
  - **场景一：首页大盘 (Dashboard)**
    - **触发时机**：用户首次安装并打开 App，进入首页时立即触发。
    - **引导锚点**：
      1. 【周期切换器】：提示用户可以在此切换查看不同月份的数据。
      2. 【可用结余卡片】：提示这里是财务推演的核心指挥中心。
      3. 【记一笔 FAB】：引导用户点击亮橙色按钮记录第一笔账单。
  - **场景二：流水列表与手势 (Transactions)**
    - **触发时机**：用户首次进入流水 Tab 页，**且**列表数据大于 0 条时触发（确保有元素可供演示滑动）。
    - **引导锚点**：
      1. 【高级筛选按钮】：提示用户可以进行精准的交叉条件搜索。
      2. 【第一条流水卡片】：高亮该卡片，明确告知用户“向左滑动此卡片可以呼出删除按钮”。
  - **场景三：周期订阅 (Subscriptions)**
    - **触发时机**：首次进入订阅页时延迟 500ms 触发。
    - **引导锚点**：
      1. 【月度均摊卡片】：向用户解释 App 会自动折算年付/季付成本，理清固定支出。
      2. 【添加订阅 FAB】：提示在录入弹窗里可以使用“立即记一笔支出”的同步开关，实现一键双写。
      3. _(动态判定)_ 【临近扣费标签】：若列表有数据，高亮右侧倒计时标签，提示 <=5 天时会标红警告；若列表为空，则静默跳过此步。
  - **场景四：设置 (Settings)**
    - **触发时机**：首次进入设置页时延迟 500ms 触发。
    - **引导锚点**：
      1. 【备份】：提示用户可以导出数据到json文件。
      2. 【恢复】：提示用户可以从json文件恢复数据。
      3. 【清空数据】：提示用户可以清空所有数据。
- **状态持久化**：指引的完成状态需保存在本地存储 (AsyncStorage) 中，保证卸载重装前不再重复弹出。
